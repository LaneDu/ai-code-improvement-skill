# AI 代码自检（金蝶云苍穹 Java）

## 用途

本技能让 AI 能够对自己生成的代码进行自检，对照已知的常见错误进行审查，修复发现的问题，并持续积累错误经验，避免重复犯错。

## 触发时机

- **自动触发**：AI 生成或大幅修改 Java 代码后（主要是金蝶云苍穹平台相关代码）
- **手动触发**：用户要求代码检查、审查、自检、质量检查时

## 自检工作流

复制以下清单并跟踪进度：

```
自检进度：
- [ ] 第1步：读取经验库
- [ ] 第2步：按所有检查分类扫描代码
- [ ] 第3步：修复所有发现的问题
- [ ] 第4步：将新错误记录到经验库
- [ ] 第5步：向用户汇报结果
```

### 第1步：读取经验库

在审查之前，先读取经验库以回顾历史错误：

```
读取文件：[skill-directory]/errors-log.md
```

确保之前遇到的错误在都有记录，避免再次犯错。

### 第2步：按检查分类扫描代码

对生成的代码逐一检查以下所有分类。详细规则和示例见 [common-mistakes.md](common-mistakes.md)。

#### A 类：逻辑与边界错误

- **A1 空值安全**：来自数据库查询、API 调用或参数的对象引用，使用前必须判空
- **A2 集合安全**：遍历前检查 `isEmpty()`；不检查 size 不要直接 `get(0)`
- **A3 类型转换**：强制转换前验证类型兼容性；使用 `DataType` 等安全转换工具
- **A4 边界条件**：循环边界、子串索引、分页偏移量必须校验
- **A5 条件逻辑**：确认 `if/else` 分支覆盖所有情况；注意反转条件和 switch 中缺失的 `break`

#### B 类：安全与性能

- **B1 禁止硬编码密钥**：API 密钥、密码、连接字符串必须来自配置
- **B2 防止 SQL 注入**：通过 ORM 使用参数化查询；禁止将用户输入拼接到 SQL
- **B3 资源管理**：数据库连接、流、锁必须在 `finally` 中关闭或使用 try-with-resources
- **B4 防止 N+1 查询**：禁止在循环内执行数据库查询；使用批量操作（`SaveServiceHelper`、`DeleteServiceHelper`）
- **B5 大数据集处理**：查询可能返回大量数据时，添加分页或限制条件

#### C 类：代码风格与规范

- **C1 命名一致性**：遵循金蝶云苍穹的实体、字段、操作命名规范
- **C2 未使用变量**：删除已声明但未使用的变量和导入
- **C3 重复代码**：将重复的逻辑提取为辅助方法
- **C4 异常处理**：禁止静默吞掉异常；必须记录日志或带上下文重新抛出
- **C5 注释质量**：为复杂业务逻辑添加有意义的注释；避免无意义注释

#### D 类：金蝶云苍穹平台特有

- **D1 插件 API 正确性**：使用正确的插件事件接口和注册模式
- **D2 ORM 用法**：正确使用 `ORM.create()`、`SaveServiceHelper`、`DeleteServiceHelper`；不要混用原生 JDBC 和 ORM
- **D3 表单操作**：确认实体/字段标识符与实际数据模型匹配
- **D4 业务规则模式**：遵循校验器、转换器和操作插件的标准模式
- **D5 事务感知**：理解哪些操作在事务内；不要在无回滚处理的情况下执行非幂等操作

### 第3步：修复所有发现的问题

对每个发现的问题：

1. 定位精确的代码位置（文件、方法、行范围）
2. 简要说明问题
3. 直接在代码中修复
4. 如果修复改变了行为，告知用户

**修复优先级**：严重（安全、数据丢失）> 高（逻辑错误、空指针崩溃）> 中（性能）> 低（风格）

### 第4步：将新错误记录到经验库

如果发现了经验库中尚未记录的新错误模式，追加到经验库：

```
读取并追加到：[skill-directory]/errors-log.md
```

追加时使用以下格式：

```markdown
### [错误编号] 简短描述
- **日期**：YYYY-MM-DD
- **分类**：A/B/C/D + 子分类
- **语言/框架**：Java / 金蝶云苍穹
- **问题描述**：出了什么问题
- **错误代码示例**：
  ```java
  // 有问题的代码
  ```
- **正确代码示例**：
  ```java
  // 修复后的代码
  ```
- **根本原因**：为什么会出这个错
- **预防规则**：以后如何避免
```

**重要**：只记录真正的新模式。如果经验库中已有类似错误，跳过记录以避免冗余。

### 第5步：汇报结果

向用户提供简洁的自检报告：

```
## 自检报告

**扫描范围**：[数量] 个文件，[数量] 个方法
**发现问题**：[数量]（严重: X，高: X，中: X，低: X）
**已修复**：[数量]
**新增经验记录**：[数量] 条新模式

### 已修复问题
1. [错误编号] - 简要描述修复内容和位置
2. ...

### 备注
- 给用户的观察或建议
```
